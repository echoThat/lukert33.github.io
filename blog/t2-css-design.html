Selector Refinement

September 28, 2014

On class and id in modern CSS

Sounds Like: Heartbreak


CSS selectors are what you use to specify style characteristics in CSS. In h1{color: beige;} h1 is the selector. We are selecting the heading-1 tag in order to bestow some some smokin' hot beige.

When you are writing a webpage using html and CSS, you quickly realize that you will need to treat two different instances of the same tag in different ways. The unordered list of vocabulary below, for instance, shows off its square bullets. The unordered list in my sidebar, however, hides them. In both cases I called on certain html functionality with the tag <ul>.d So how do I tell your browser to render them differently? I do so using class="" and id="". These are "selectors," both of which are more specific than a tag selector like <ul>. Let's go ahead and lay out the vocabulary I mentioned a moment ago:

Vocabulary

  • tag - an html element with built-in behavior. Head, body, heading, img, div, article, ordered list...they're all tags.
  • class - one step more specific than tag. You specify a class selector on a tag when you will have several elements in your site that need to have the same style attributes. When I write <ul class="bulleted"></ul> I am adding an unordered list for which I intend to display bullets. I'll have other unordered lists where I hide them, and several instances of both over my whole site.
  • id - one step more specific than class. Use an id when you are only going to apply a style to one specific instance of am html tag. In <ul id="shark-bullets"></ul> I am adding the one unordered list I will have on the site that features animated chompy shark GIFS for bullets.

So as you can see, all of these are selectors. You should treat id as more specific than class and class as more specific than tag. Html elements will all inherently have a tag selector. You can also add as many class selectors or id selectors as you need. <ul class="horizontal" id="shark-bullets"></ul> imports styling that I'll use elsewhere on my site to make that chompy shark list render horizontally, rather than the default of vertical.

Selectors are by definition NOT mutually exclusive. The <ul> tag passes down all of the style characteristics to <ul class="horizontal" id="shark-bullets"></ul>. That shark bullet list, however, has the authority to override any characteristics of a default unordered list that you specify with your CSS selector definition in .horizontal{display:inline;} and #shark-bullets{list-style-image:chompy_shark.gif;}

It's important to note that when a user's browser renders your website, it will give priority to the most specific selector. The most specific selector should be on the right, closest to the style parameters being passed to it. So when you have a large class and a more specific subclass within that, you should write it like so: .top-bar .horizontal-list{background-color:fuschia;} because while there are several top-bar elements we are applying position styling so, there is only one specific element we want to color fuschia. When you have an element with an id, there should only be one of them on your whole site. So that means that you should only call the id in your CSS, as in #shark-bullets{list-style-image:chompy_shark.gif;}. You can have multiple elements that have the same id, and a browser will render them with the CSS style applied to that id, but if you do that, you should be creating a class rather than an id, for purposes of readability, and until 2011, rendering efficiency.

That's the simple truth of the tag > class > id hierarchy. They keep your stylesheet clean, and allow others to read your code and competently apply your style.
