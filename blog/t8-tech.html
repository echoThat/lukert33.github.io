Regular Expressions (aka Regex, Regexp)

Nov 9, 2014

Regular expressions describe patterns of characters. Commonly referred to with the shorthand regexp, they are an integral part of most common programming languages. Most languages use similar commands to call regexps.

Regexps can be very high level, for example / \d/ describes any digit, and / \s/ descibres any whitespace (spaces, tab indents, line breaks). They can also be tasked for finding very specific patterns. /cobra/ looks for the string of letters, c-o-b-r-a, in exactly that arrangement. /[aeiouy3]/ employs brackets to indicate a set of characters. If the string being searched contains any one of those characters (the vowels, y, and--for whatever reason, the digit 3), it will match the regexp.

In Ruby, regexps are objects; instances of their very own Class,

Regexp.

, i.e. /\d/.class #=>Regexp < Object . As you've seen in my examples so far, they can be denoted with /.../. You can also use %r{...}. Both of those represent the Regexp literals.

To be very meta about it, let's say you want to search a document for the string describing the Regexp literal, %r{...} to readers. In a string, when you want to write quotation marks, you have to escape them (because they mark the string literal, "". So you deploy an apostrophe with: puts "what\'s up?". You use the backslash as an escape character. In Regexp's, the backslash is also the escape character. So to find the index position of the beginning of the first Regexp literal in document you'd use:

document =~ /\%\{/ 26 #this being the index position where the first Regexp opens in the string.

To help you get started, here is a list of the common character classes, search for in Regexps by escaping with the backslash:

  • \w = word character (alphanumerics and underscores)
  • \W = non-word character
  • \d = digits
  • \D = non-digits
  • \s = whitespace
  • \S = non-whitespace
  • \b = word boundaries
  • \B = non-word boundaries
  • \n = newline

This should allow you to dip your toes into Regexp use. As you work, you may want to refer to this concise guide to Perl's special Regexp characters, which is what many popular languages like Ruby, JavaScript, and Python use.
